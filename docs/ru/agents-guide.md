# Настройка агентов

Полное описание параметров агентов и их конфигурации.

## Типы адаптеров

Адаптер определяет, как агент выполняет задачи.

| Адаптер | Описание | Когда использовать |
|---------|----------|--------------------|
| `openclaw_gateway` | Работа через NemoClaw-песочницу | Основной режим -- безопасное выполнение в sandbox |
| `claude_local` | Claude Code локально на сервере | Разработка и отладка без sandbox |
| `codex_local` | OpenAI Codex локально | Альтернативная модель без sandbox |
| `process` | Произвольный процесс | Кастомные скрипты и инструменты |
| `http` | Внешний HTTP-сервис | Интеграция со сторонними сервисами |

Для продакшена рекомендуется `openclaw_gateway` -- агент работает в изолированной среде с контролем сетевого доступа.

## Основные поля агента

| Поле | Тип | Описание |
|------|-----|----------|
| `name` | string | Отображаемое имя агента |
| `slug` | string | Уникальный идентификатор (латиница, дефисы) |
| `role` | string | Роль: `ceo` (руководитель) или `worker` (исполнитель) |
| `title` | string | Должность для отображения в UI |
| `reportsTo` | string/null | Slug руководителя (null для CEO) |
| `capabilities` | string | Навыки через запятую |
| `adapterType` | string | Тип адаптера (см. таблицу выше) |
| `model` | string | Модель LLM (зависит от адаптера) |

## Heartbeat-политика

Heartbeat -- механизм, по которому агент проверяет наличие задач и выполняет их.

```json
{
  "heartbeat": {
    "intervalSec": 300,
    "cooldownSec": 15,
    "wakeOnAssignment": true,
    "wakeOnDemand": true,
    "maxConcurrentRuns": 3
  }
}
```

| Параметр | Описание | Значение по умолчанию |
|----------|----------|-----------------------|
| `intervalSec` | Интервал проверки задач (секунды) | 300 (5 мин) |
| `cooldownSec` | Пауза после завершения запуска | 15 |
| `wakeOnAssignment` | Проснуться при назначении задачи | true |
| `wakeOnDemand` | Проснуться по запросу | true |
| `maxConcurrentRuns` | Максимум одновременных запусков | 3 |

Рекомендации:
- Управляющие агенты (CEO): `intervalSec: 600`, `maxConcurrentRuns: 2`
- Активные исполнители: `intervalSec: 300`, `maxConcurrentRuns: 3-5`

## Управление бюджетом

Бюджет задаётся в поле `budgetMonthlyCents` (центы за месяц):

| Значение | Бюджет |
|----------|--------|
| 1000 | $10/мес |
| 5000 | $50/мес |
| 10000 | $100/мес |

При превышении бюджета агент приостанавливается до начала следующего месяца. Отслеживайте расходы на странице агента.

## Стратегии рабочей области

Определяют, где агент выполняет код.

| Стратегия | Описание |
|-----------|----------|
| `project_primary` | Работа в основной директории проекта (по умолчанию) |
| `git_worktree` | Изолированный git worktree для каждой задачи |

Стратегия `git_worktree` полезна, когда несколько агентов работают над разными задачами одного репозитория -- каждый получает отдельную ветку.

```json
{
  "workspaceStrategy": {
    "type": "git_worktree",
    "branchTemplate": "{{issue.identifier}}"
  }
}
```

## Сетевые политики (NemoClaw)

При использовании адаптера `openclaw_gateway` агент работает в песочнице NemoClaw со строгой сетевой политикой. По умолчанию разрешены только:

- `api.anthropic.com` -- API Claude
- `integrate.api.nvidia.com` -- NVIDIA inference
- `github.com`, `api.github.com` -- GitHub
- `registry.npmjs.org` -- npm (только чтение)

Все остальные запросы блокируются. Оператор получает уведомление и может одобрить или отклонить запрос в реальном времени через TUI.

Для изменения политики отредактируйте файл:

```
nemoclaw-blueprint/policies/openclaw-sandbox.yaml
```

И примените через:

```bash
nemoclaw onboard
```

Или динамически для работающего sandbox:

```bash
openshell policy set <файл-политики>
```

## Подключение OpenClaw к Paperclip

Для агентов с адаптером `openclaw_gateway` нужно подключить OpenClaw к Paperclip:

1. Откройте настройки компании
2. Перейдите в **Доступ**
3. Создайте invite-ссылку
4. OpenClaw использует эту ссылку для авторизации в Paperclip

После подключения агент начнёт получать задачи через heartbeat.

# Создание первой компании

Пошаговое руководство: от пустой системы до работающей команды AI-агентов.

## Открыть Paperclip

Перейдите в браузере по адресу:

```
http://localhost:3100
```

## Создание компании

1. Нажмите **Новая компания**
2. Введите название (например, "Моя компания")
3. Добавьте описание -- оно поможет агентам понимать контекст работы
4. Нажмите **Создать**

## Импорт из шаблона

Вместо ручной настройки можно импортировать готовый шаблон с командой агентов. Доступные шаблоны:

| Шаблон | Файл | Агенты |
|--------|------|--------|
| Маркетинговое агентство | `marketing-agency.json` | 4: директор, контент, SMM, аналитик |
| Разработка ПО | `software-dev.json` | 4: CTO, backend, frontend, QA |
| Поддержка клиентов | `customer-support.json` | 3: руководитель, оператор, аналитик |

Импорт через API:

```bash
curl -X POST http://localhost:3100/api/companies/import \
  -H "Content-Type: application/json" \
  -d @paperclip/templates/marketing-agency.json
```

Подробнее -- в [руководстве по шаблонам](templates.md).

## Создание агентов вручную

Если шаблон не подходит, создайте агентов самостоятельно.

### Добавление агента

1. Откройте компанию
2. Нажмите **Добавить агента**
3. Заполните поля:

| Поле | Описание | Пример |
|------|----------|--------|
| Имя | Отображаемое имя | "Менеджер проекта" |
| Роль | `ceo` или `worker` | `worker` |
| Тип адаптера | Способ выполнения задач | `openclaw_gateway` |
| Capabilities | Навыки через запятую | `planning, coordination` |
| Подчиняется | Slug руководителя | `direktor` |

### Выбор адаптера

Для работы в песочнице NemoClaw используйте `openclaw_gateway` -- это основной тип.

Другие варианты:
- `claude_local` -- Claude Code локально на сервере
- `codex_local` -- OpenAI Codex локально
- `process` -- произвольный процесс

### Настройка инструкций

Каждый агент получает системные инструкции в формате markdown (файл `AGENTS.md`). Опишите:
- Роль и обязанности
- Стиль работы
- Ограничения и правила

### Установка бюджета

Бюджет задаётся в центах за месяц. Например, `5000` = $50/месяц. При превышении бюджета агент приостанавливается.

## Создание цели

1. Откройте компанию
2. Перейдите в раздел **Цели**
3. Нажмите **Новая цель**
4. Укажите название и описание

Цели задают стратегическое направление для агентов.

## Создание проекта

1. Перейдите в раздел **Проекты**
2. Нажмите **Новый проект**
3. Укажите название и привяжите к цели

Проекты группируют задачи в рамках одной цели.

## Назначение задачи

1. Откройте проект
2. Создайте задачу (issue)
3. Назначьте исполнителя -- выберите агента
4. Агент получит задачу при следующем heartbeat

## Мониторинг

### Запуски (Runs)

Каждое выполнение задачи агентом -- это запуск. На странице агента видны:
- Статус текущего запуска
- История запусков
- Логи выполнения

### Расходы

На странице агента отображается:
- Текущие расходы за месяц
- Оставшийся бюджет
- Стоимость каждого запуска

Если расходы растут слишком быстро, уменьшите бюджет или увеличьте интервал heartbeat.
